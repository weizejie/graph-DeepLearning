### 1.3 RouteNet 的核心创新与局限

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                           RouteNet 的核心创新                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  ✓ 创新1: 异构超图建模                                                            │
│  ─────────────────────                                                              │
│  • 统一表示 Path、Link、Queue 三种不同语义的网络元素                              │
│  • 通过关系矩阵实现拓扑不变性                                                      │
│                                                                                     │
│  ✓ 创新2: GRU 消息传递                                                            │
│  ─────────────────────                                                              │
│  • 迭代 8 次的状态更新，捕获网络中的迭代收敛现象                                  │
│  • 路径 → 队列 → 链路的循环依赖通过 GRU 解决                                      │
│                                                                                     │
│  ✓ 创新3: 拓扑零样本泛化                                                          │
│  ─────────────────────                                                              │
│  • 能够对训练时未见过的网络拓扑进行预测                                            │
│  • 基于归纳偏置实现                                                                │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                           RouteNet 的潜在局限                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  ⚠ 局限1: 缺乏拓扑结构感知                                                        │
│  ──────────────────────────                                                        │
│  • 当前方法主要关注路径和队列的数值特征                                            │
│  • 没有显式建模网络的拓扑结构（如环路、连通分量）                                  │
│  • 可能丢失网络的"结构指纹"                                                        │
│                                                                                     │
│  ⚠ 局限2: 时序建模缺失                                                            │
│  ──────────────────────                                                            │
│  • 当前处理的是静态网络快照                                                        │
│  • 没有考虑流量随时间的动态变化                                                    │
│  • 网络攻击、流量突发等场景难以建模                                                │
│                                                                                     │
│  ⚠ 局限3: 流量模型类型未充分利用                                                  │
│  ─────────────────────────────                                                      │
│  • 7 种流量模型 (EXP/UNIF/NORMAL/ONOFF/PPBP/TRACE) 编码为 one-hot                 │
│  • 没有利用流量模型的结构差异进行特征增强                                          │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

│  │   • 如: 核心-边缘拓扑                                                         │   │

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  ★ RouteNet 中 DoS 的具体应用场景                                                  │
│                                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────────────┐   │
│  │                                                                             │   │
│  │   应用1: 拓扑结构编码为拓扑嵌入                                                │   │
│  │   ─────────────────────────────                                               │   │
│  │                                                                             │   │

│  │                                                                             │   │

### 6.2 实现代码框架

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                        RouteNet-H 实现代码框架                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  # 1. 拓扑描述符计算                                                               │
│                                                                                     │
│  def compute_topology_descriptors(topology_nx):                                   │
│      """                                                                         │
│      计算网络的拓扑描述符                                                          │
│      """                                                                         │
│                                                                                     │
│      # 构建邻接矩阵                                                                │
│      A = nx.adjacency_matrix(topology_nx).toarray()                               │
│      n = A.shape[0]                                                               │
│                                                                                     │
│      # 计算度矩阵和归一化拉普拉斯                                                  │
│      D = np.diag(A.sum(axis=1))                                                   │
│      D_inv_sqrt = np.diag(1.0 / np.sqrt(np.diag(D) + 1e-10))                   │
│      L = np.eye(n) - D_inv_sqrt @ A @ D_inv_sqrt                                  │
│                                                                                     │
│      # 特征值分解 (小规模网络)                                                    │
│      eigenvalues, _ = np.linalg.eigh(L)                                           │
│      eigenvalues = np.sort(eigenvalues)                                            │
│                                                                                     │
│      # DoS 谱密度直方图 (4 bins)                                                  │
│      psi = np.histogram(eigenvalues, bins=4, range=(0, 2), density=True)[0]    │
│                                                                                     │
│      # 代数连通度 (最小非零特征值)                                                 │
│      lambda_2 = eigenvalues[1] if n > 1 else 0                                    │
│                                                                                     │
│      # Betti 数 (可选, 计算代价较高)                                              │
│      try:                                                                        │
│          betti = compute_betti_numbers(topology_nx)  # 使用 pyflagser           │
│          beta_0, beta_1 = betti[0], betti[1]                                    │
│      except:                                                                      │
│          beta_0, beta_1 = 1, 0  # 默认值                                          │
│                                                                                     │
│      # 返回拓扑描述符向量                                                          │
│      return [beta_0, beta_1, *psi, lambda_2]  # 长度 7                          │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                     │
│  # 2. 集成到 RouteNet                                                              │
│                                                                                     │
│  class RouteNet_H(keras.Model):                                                   │
│      def __init__(self, ...):                                                     │
│          super().__init__()                                                        │
│          # ... RouteNet 原有组件 ...                                              │
│                                                                                     │
│          # 新增: 拓扑嵌入层                                                        │
│          self.topo_embedding = Sequential([                                       │
│              Input(shape=7),  # [β₀, β₁, ψ₁, ψ₂, ψ₃, ψ₄, λ₂]                  │
│              Dense(32, activation='relu'),                                        │
│              Dense(32, activation='relu')                                        │
│          ])                                                                       │
│                                                                                     │
│      def call(self, inputs):                                                      │
│          # ... RouteNet 原有前向传播 ...                                          │
│                                                                                     │
│          # 新增: 获取拓扑并计算描述符                                              │
│          topo_descriptors = compute_topology_descriptors(topology)                │
│          topo_embed = self.topo_embedding(topo_descriptors)                        │
│                                                                                     │
│          # 拼接: 原始嵌入 + 拓扑嵌入                                              │
│          enhanced_embed = tf.concat([original_embed, topo_embed], axis=-1)        │
│                                                                                     │
│          # ... 后续 GRU 消息传递 ...                                              │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘
```
